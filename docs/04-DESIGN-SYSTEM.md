# Design System Direction

## Brand Personality

CloudFolio should feel senior, technical, reliable, and internationally professional.

The product should communicate:

- Calm confidence
- Engineering precision
- Operational maturity
- Clear decision-making
- Trustworthiness
- Global career readiness

It should avoid looking like a generic resume template or a visual-heavy landing page with weak substance.

## Color Palette

The future palette should support a serious engineering product:

- Primary: deep neutral or technical blue for trust and infrastructure context
- Secondary: restrained green or teal for reliability, systems health, and cloud operations
- Accent: warm highlight for calls to action and key achievements
- Neutral scale: strong grays for documentation, diagrams, and dense technical content
- Semantic colors: success, warning, error, and info states with accessible contrast

The palette should work in both light and dark modes without relying on color alone to communicate meaning.

## Typography

Typography should prioritize readability and professional tone:

- Clear sans-serif typeface for interface and body text
- Monospace typeface for commands, infrastructure terms, configuration references, and technical snippets when implementation begins
- Strong heading hierarchy for scanning
- Comfortable line length for technical writing
- Avoid decorative fonts

## Spacing System

Use a consistent spacing scale to support predictable layouts:

- Tight spacing for related metadata and tags
- Moderate spacing for cards, lists, and content groups
- Generous spacing for major page sections
- Consistent alignment across page templates

Dense technical pages should remain scannable without becoming crowded.

## Layout Principles

- Prioritize content clarity over decorative layout.
- Use responsive grids for project, blog, certification, and case study indexes.
- Keep recruiter workflows direct and low-friction.
- Place strongest professional signal above the fold.
- Make technical evidence easy to browse and cross-reference.
- Use diagrams and structured sections for complex architecture content.

## Dark Mode Strategy

Dark mode should be treated as a first-class experience for technical users.

Requirements:

- Maintain WCAG-aligned contrast.
- Avoid pure black backgrounds for long-form reading.
- Preserve diagram readability.
- Use semantic tokens rather than one-off colors.
- Ensure code, metadata, links, and cards remain distinguishable.

## Accessibility Principles

- Semantic page structure and heading order
- Keyboard-accessible navigation and controls
- Visible focus states
- Sufficient color contrast
- Descriptive link and button labels
- Reduced-motion support
- Screen-reader-friendly form labels and error states
- Accessible diagrams with summaries or alt text

## Animation Philosophy

Animation should support comprehension, not distract from credibility.

Use animation for:

- Subtle transitions
- Loading and state changes
- Timeline progression
- Diagram focus states
- Microinteractions that confirm user actions

Avoid excessive motion, autoplay-heavy experiences, or animations that slow down recruiter workflows.

# SEO Strategy

## Metadata

Every public page should define:

- Title
- Meta description
- Canonical URL
- Page type
- Author
- Relevant keywords
- Language
- Last updated date when appropriate

Page titles should prioritize name, role, and topic clarity.

Example title patterns:

- Narendra Pratap Singh | Senior Cloud Infrastructure Engineer
- AWS and Kubernetes Projects | CloudFolio
- SRE Case Studies | Narendra Pratap Singh

## Open Graph

Open Graph metadata should support high-quality sharing on LinkedIn, Slack, WhatsApp, and other platforms.

Include:

- `og:title`
- `og:description`
- `og:type`
- `og:url`
- `og:image`
- `og:site_name`

Images should be professional, legible, and tailored for major page types.

## Structured Data

Future structured data should cover:

- Person
- ProfilePage
- WebSite
- BlogPosting
- CreativeWork for case studies
- EducationalOccupationalCredential for certifications when appropriate
- BreadcrumbList

Structured data must be truthful and aligned with visible page content.

## Sitemap

The platform should provide an XML sitemap once application routing exists.

Include:

- Core pages
- Project detail pages
- Case study detail pages
- Blog posts
- Architecture pages
- Certification pages

Exclude low-value, duplicate, private, or draft content.

## Robots

Robots configuration should:

- Allow indexing of public portfolio content
- Block draft or private paths if they exist
- Reference the XML sitemap
- Avoid blocking assets required for rendering

## Personal Branding Strategy

CloudFolio should position Narendra as:

- A senior cloud and infrastructure engineer with 14+ years of experience
- A practical operator of production systems
- A clear technical communicator
- A candidate prepared for international senior roles
- A professional who connects engineering decisions to business outcomes

The SEO strategy should support discovery while preserving authenticity and avoiding keyword stuffing.

# User Flows

## Recruiter Journey

Goal: Determine fit quickly and initiate contact.

Path:

1. Home
2. Recruiter Hub
3. Role Fit
4. Resume
5. Location Preferences
6. Contact

Expected outcome:

- Understands target roles, seniority, skills, countries, and contact path.
- Can download or request the latest resume.
- Can decide whether to move Narendra into a hiring process.

## Hiring Manager Journey

Goal: Validate senior-level engineering capability.

Path:

1. Home
2. Case Studies
3. Architecture Gallery
4. Projects
5. Resume
6. Contact

Expected outcome:

- Sees evidence of cloud, DevOps, platform, SRE, and AWS experience.
- Reviews tradeoffs, decisions, and measurable outcomes.
- Identifies interview topics and role alignment.

## Engineer Journey

Goal: Inspect technical depth and implementation thinking.

Path:

1. Blog
2. Architecture Gallery
3. Projects
4. Case Study Detail
5. GitHub or Open Source Links

Expected outcome:

- Finds technical explanations, diagrams, and engineering rationale.
- Understands how Narendra approaches infrastructure, reliability, automation, and operations.
- Can evaluate practical skill depth beyond keywords.

## Visitor Journey

Goal: Learn who Narendra is and what CloudFolio represents.

Path:

1. Home
2. About
3. Timeline
4. Featured Projects
5. Contact

Expected outcome:

- Understands the professional story.
- Finds clear navigation to deeper content.
- Can contact or share the profile if relevant.

## Navigation Principles

- Every major page should offer a path to the Recruiter Hub.
- Technical pages should connect to related projects, case studies, and architecture diagrams.
- Resume pages should connect to evidence, not only list skills.
- Contact should remain accessible from primary navigation and recruiter-focused pages.

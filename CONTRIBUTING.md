# Contributing to CloudFolio

Thank you for your interest in contributing to CloudFolio.

CloudFolio is being developed as a professional engineering portfolio and open-source learning project. Contributions should support clarity, maintainability, and long-term project quality.

## Current Project Stage

The repository is currently in its foundation stage. Application code, package configuration, framework setup, Docker files, and CI workflows have not been added yet.

Please avoid introducing implementation code until the relevant milestone is defined.

## Contribution Principles

- Keep changes focused and easy to review.
- Prefer clear documentation over implicit assumptions.
- Follow existing repository structure and naming conventions.
- Explain meaningful tradeoffs in issues, pull requests, or `DECISIONS.md`.
- Do not add dependencies, frameworks, generated files, or tooling without a documented reason.

## Development Workflow

1. Open or reference an issue for non-trivial changes.
2. Create a focused branch from `main`.
3. Make the smallest coherent change.
4. Update documentation when behavior, workflow, or project direction changes.
5. Open a pull request with a clear summary and validation notes.

## Pull Request Guidelines

Pull requests should include:

- A concise description of the change
- The reason the change is needed
- Any relevant screenshots, logs, or notes
- Testing or validation performed
- Any follow-up work that remains

## Documentation Standards

Documentation should be:

- Clear and professional
- Accurate for the current project state
- Free of unnecessary placeholders
- Updated when decisions or workflows change

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
